## ARM, FOOT 모터 조립, 초기 상태
- 본체 - ARM 모터 - ARM Connector - Foot Connector - FOOT 모터 - FOOT 연결
- 본체와 4개 ARM Connector를 연결하는 모터는 혼의 방향이 윗쪽이다.
- 모터의 초기 각도는 90도로 설정하고 ARM는 본체의 대각선상에 위치하도록 조립했다.
- 각 모터의 이름은 A(front-left), B(front-right), C(back-left), D(back-right) 이다.
- ARM 모터는 ARM Connector와 연결되어 있다.
- ARM 모터축이 반시계 방향 회전하므로 ARM Connector도 반시계 방향으로 회전한다.
- 4개의 ARM 모터축이 모두 동일 방향으로 회전하므로 이것을 고려해서 coding해야한다.
- 각 ARM Connector에 연결된 FOOT Connector는 모터의 초기 각도를 120도로 설정하고 FOOT 본체,ARM과 평행하도록 조립했다.(수평)
- Foot Connector에 Foot 모터가 고정되어 있고, Foot은 모터 본체에 고정되어 있다.
  따라서 Foot은 시계 방향 회전한다.

## 좌표계 정의 (2026.04.05 추가)

### 논리각 (Logical Angle) — 월드 기준
- 모든 소프트웨어(main.cpp, walk_sim)에서 공통으로 사용하는 기준 좌표계
- 0° = 전방, 90° = 수평(팔이 옆으로), 180° = 후방
- CCW = 양수, CW = 음수
- 서보 제어 시에만 toServo() 변환 함수를 통해 물리각으로 변환

### 물리각 (Physical Angle) — 서보 모터 기준
- 서보 모터 하드웨어 제어각 (0~180°)
- toServo() 함수를 통해서만 접근, 외부에서 직접 사용 금지

### ARM 논리각 변환 (논리각 → 물리각)
- 몸체는 사각형, 각 모서리를 지나는 대각선 방향이 phyStl=90° (논리각 기준 중립)
- 앞다리(A/B) 수평 = 물리각 135°/45°, 뒷다리(C/D) 수평 = 물리각 45°/135°

| 다리 | 위치 | 논리각 범위 | 수평(물리각) | 변환식 |
|------|------|-------------|-------------|--------|
| A | 앞왼 | -45° ~ 135° | 135° | phy = logical + 45 |
| B | 앞오 | -135° ~ 45° | 45°  | phy = logical + 135 |
| C | 뒤왼 | 45° ~ 225°  | 45°  | phy = logical - 45 |
| D | 뒤오 | -225° ~ -45° | 135° | phy = logical + 225 |

- ARM_LIMIT: 기구 안전을 위해 양 끝단에서 여유각 설정 (기본 15°)
- armOffset[4]: 모터별 조립 공차 보정용 개별 offset (main.cpp만 해당)

### FOOT 논리각 변환 (논리각 → 물리각)
- 변환식: phy = 210 - logical (A~D 동일)
- FOOT_LIMIT: 양 끝단 안전 여유각 (기본 10°)

| 논리각 | 물리각 | 상태 |
|--------|--------|------|
| 210°   | 0°     | 물리 한계 (limit으로 차단) |
| 180°   | 30°    | 발 최대 위 |
| 170°   | 40°    | Fdw (서있는 자세) |
| 135°   | 75°    | Stl (대기 자세) |
| 120°   | 90°    | Fup (몸 낮게) |
| 90°    | 120°   | 수평 |
| 30°    | 180°   | 수직 (최대 아래) |

## 관절 동작 범위 (물리각 기준)

### Arm
- 개별 각도는 0도~180도이다. 설계/조립 공차 고려하면 5도~175도이다.
- 인접한 팔이 서로 가까워지는 경우 고려하면 25도 ~ 155도이다.
- 방향 : 방향(위에서 내려볼때 팔 각도-전방 0도, 제어각도)
  - A: 전방(0,45), 대기(45,90), 좌측(90,135)
  - B: 전방(0,135), 대기(-45,90), 우측(-90,45)
  - C: 후방(180,135), 대기(135,90), 좌측(90,45)
  - D: 후방(180,45), 대기(45,90), 우측(-90,135)

### Foot
- 개별 각도는 0도~180도이다. 설계/조립 공차 고려하면 5도~175도이다.
- 인접한 구조물과 간섭은 없다.
- 방향 : A~D 동일, 방향(수직면 기준 정면관측각도, 제어각도)
  - 위-대각(30,180), 수평(90,120), 대기(150,60)
    아래-수직(180,30), 아래-몸쪽(210,0도)

## WALK 동작
- 정적(Crawl) 방식으로 구현
- 현재 제어보드와 배터리가 뒷쪽에 있어서 무게중심이 약간 뒷쪽에 있다
- 앞:뒤 = 45:55
- 코드에서 go_ahead 함수와 같이 노젓는 방법으로 전진하지 않는다.
- BCD고정-A보행-ABC고정-D보행-ACD고정-B보행-ABD고정-C보행
- 보행 시 앞뒤로 넘어지지 않도록 무게 중심을 잡아야한다.
- 무게 중심을 잡기 위해 보행 발을 들기 전에 사전 처리가 필요하다. (CoG pre-shift)
